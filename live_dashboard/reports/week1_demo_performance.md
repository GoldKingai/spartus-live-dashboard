## Normalizer Incident (March 4)

- **Root cause:** NY gold crash ($5370->$5043) contaminated adaptive rolling z-score buffers
- **Effect:** 15/38 features drifted, conviction collapsed to 0.05
- **Fix:** Switched to frozen normalization mode (static baseline from training)
- **Post-fix conviction:** Mean 0.43 (up from 0.05 during collapse)
- **Lesson:** RL policies trained under fixed distributions must use frozen normalization in production

## Recommendations for Next Model (Priority Order)

### 1. Raise live conviction threshold (0.15 -> 0.30+)
Conviction < 0.3 trades are net zero or negative. Only conv > 0.5 is reliably profitable.

### 2. Fix LONG bias
63% of trades are LONG, and LONGs lose money. Consider rebalancing training data, adding directional accuracy rewards, or training with more bear-market episodes.

### 3. Improve regime detection
The model cannot detect trending sell-offs. It re-enters LONG repeatedly into a falling market. Memory features need to capture "recent direction accuracy" more aggressively.

### 4. Increase minimum hold time (3 -> 6 bars)
76% of trades exit at the minimum hold. The model hasn't learned patience. Combined with 1:1 R:R, this requires >50% accuracy to profit.

### 5. NY session handling
NY session (14:00 UTC especially) is catastrophic. Calendar features should be verified working. Consider a specific NY volatility penalty.

### 6. Improve trailing SL
24 losing trades had >2pt favorable excursion before reversing. The sl_adjustment action output is barely used. The trailing SL needs to be more aggressive at locking profit.
